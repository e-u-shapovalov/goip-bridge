# Как скачать goip-bridge с GitHub и не перепутать с исходным кодом

Этот файл для тех, кто не знает Git, GitHub, Go и не хочет собирать программу вручную.

**Главное правило:** обычному пользователю нужен готовый файл из **Releases**, а не кнопка `Code -> Download ZIP`.

## Что скачивать

Скачивайте готовый бинарный файл из раздела **Releases -> Assets**:

- `goip-bridge`
- `goip-bridge-linux-amd64`
- `goip-bridge-linux-amd64.tar.gz`

Название может немного отличаться, но в нем должно быть `goip-bridge`. Если есть несколько вариантов, для обычного Linux-сервера x86-64 выбирайте файл с `linux-amd64`.

## Что не скачивать

Не скачивайте эти файлы, если хотите просто запустить программу:

- `Source code (zip)`
- `Source code (tar.gz)`
- архив из кнопки `Code -> Download ZIP`

Это исходный код. Он нужен разработчикам, а не пользователям, которые хотят установить и запустить готовую программу.

## Где находится Releases

На странице проекта GitHub обычно показывает блок **Releases** справа от списка файлов. Если вы открыли репозиторий с телефона или узкого экрана, блок может быть ниже.

Порядок действий:

1. Откройте страницу проекта на GitHub.
2. Найдите **Releases**.
3. Нажмите **Latest**.
4. Прокрутите страницу релиза до блока **Assets**.
5. Скачайте готовый файл `goip-bridge...`.

Если блока **Assets** нет или там только `Source code`, значит готовый файл к релизу еще не прикрепили.

## Как запустить после скачивания

Готовый релиз рассчитан на Linux x86-64 / amd64.

1. Скопируйте скачанный файл на сервер, где будет работать bridge.
2. Если это архив `.tar.gz`, распакуйте его:

```sh
tar -xzf goip-bridge-linux-amd64.tar.gz
```

3. Положите рядом `config.json`. Шаблон лежит в [config.example.json](config.example.json).
4. Откройте `config.json` и поменяйте минимум `http_token`.
5. Разрешите запуск:

```sh
chmod +x goip-bridge
```

6. Запустите:

```sh
./goip-bridge -config config.json
```

Если файл называется `goip-bridge-linux-amd64`, можно либо запускать его под этим именем, либо переименовать:

```sh
mv goip-bridge-linux-amd64 goip-bridge
chmod +x goip-bridge
./goip-bridge -config config.json
```

## Что настроить в GoIP

В веб-интерфейсе GoIP для нужного канала найдите настройки **SMS** и укажите:

```text
SMS Server IP: IP-адрес Linux-сервера с goip-bridge
SMS Server Port: 44444
Client ID: как настроено на GoIP, например Go1
Password: как настроено на GoIP
```

После этого GoIP должен начать отправлять keepalive на bridge. Проверка:

```sh
curl -H "Authorization: Bearer CHANGE_ME" http://127.0.0.1:8080/lines
```

Если вместо `CHANGE_ME` вы указали другой `http_token`, используйте свой токен.

## Частые ошибки при скачивании

### Я скачал ZIP, но не понимаю, что запускать

Скорее всего, это исходный код. Вернитесь в **Releases -> Assets** и скачайте готовый файл `goip-bridge...`.

### В Releases нет готового файла

Значит релиз еще не подготовлен для обычных пользователей. Нужно собрать программу из исходников или попросить автора добавить бинарник в GitHub Release.

### Windows говорит, что файл не запускается

Текущий готовый релиз описан для Linux x86-64. Запускайте `goip-bridge` на Linux-сервере, VPS, мини-ПК или другой Linux-машине в сети с GoIP.

### Браузер предупреждает о неизвестном файле

Для open-source бинарников это нормально, если файл новый и не имеет репутации у браузера. Скачивайте программу только со страницы проекта автора и сверяйте название релиза.

## English quick note

For end users: download the ready binary from **GitHub Releases -> Assets**. Do not download `Source code` and do not use `Code -> Download ZIP` unless you want to build from source.
